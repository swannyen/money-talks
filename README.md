# Money Talks

A personal investment dashboard for tracking **multiple portfolios** in one place. Record buy/sell/dividend/cash transactions, store them in **PostgreSQL (Supabase)**, and view holdings, dividends, realised P/L, and total returns in a **Streamlit** app.

## What you get

| Tab | What it shows |
|-----|----------------|
| **Overview** | Portfolio-level value, cost, unrealised P/L, and allocation pie chart |
| **Holdings** | Open positions (WAC cost basis), live prices, unrealised P/L |
| **Dividends** | Dividend history by year/ticker and dividend recovery vs cost basis |
| **Realised P/L & Returns** | Total return per ticker and realised P/L on sells |
| **Raw Transactions** | View, add, edit, and delete transactions in the database |

Supported transaction actions: `BUY`, `SELL`, `DIVIDEND`, `DEPOSIT`, `FEE`.

## Quick start (local)

### 1. Clone and install

```bash
git clone <your-fork-or-repo-url>
cd money-talks
pip install -r requirements.txt
```

Python 3.11+ recommended.

### 2. Create a Supabase database

1. Create a free project at [supabase.com](https://supabase.com).
2. In the SQL editor, create the transactions table:

```sql
CREATE TABLE transactions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  date TIMESTAMP,
  portfolio TEXT,
  ticker TEXT,
  asset_name TEXT,
  asset_class TEXT,
  currency TEXT,
  action TEXT,
  quantity INTEGER,
  value DOUBLE PRECISION,
  value_base DOUBLE PRECISION,
  price_per_unit DOUBLE PRECISION,
  year INTEGER
);
```

3. Get your **connection string** from **Project Settings → Database → Connection pooling → Transaction mode** (port `6543`).

   Use the **pooler** host (`aws-*-*.pooler.supabase.com`), not the direct `db.*.supabase.co` URL. The direct host is often IPv6-only and fails on many home networks with a DNS error.

   Format:

   ```text
   postgresql://postgres.<project-ref>:<password>@aws-0-<region>.pooler.supabase.com:6543/postgres
   ```

### 3. Configure secrets

```bash
cp .streamlit/secrets.toml.example .streamlit/secrets.toml
```

Edit `.streamlit/secrets.toml`:

```toml
APP_PASSWORD = "choose-a-strong-password"

DATABASE_URL = "postgresql://postgres.<project-ref>:<password>@aws-0-<region>.pooler.supabase.com:6543/postgres"
```

Do **not** commit `secrets.toml` (it is gitignored).

Optional: set portfolio names, currencies, and base currency in `.streamlit/secrets.toml` or a `.env` file:

```toml
PORTFOLIOS = "IBKR,CPF,Robinhood"
CURRENCIES = "SGD,USD,HKD,EUR,JPY"
BASE_CURRENCY = "SGD"
```

Comma-separated strings work in secrets and environment variables. In Streamlit Cloud secrets you can also use TOML arrays:

```toml
PORTFOLIOS = ["IBKR", "CPF", "Robinhood"]
```

### 4. Run the app

```bash
streamlit run app.py
```

Log in with `APP_PASSWORD`, then add transactions from **Raw Transactions → Add Transaction**.

## Deploy on Streamlit Cloud

1. Push this repo to GitHub.
2. Create a new app at [share.streamlit.io](https://share.streamlit.io) pointing at `app.py`.
3. Under **App settings → Secrets**, paste:

```toml
APP_PASSWORD = "your-strong-password"
DATABASE_URL = "postgresql://postgres.<project-ref>:<password>@aws-0-<region>.pooler.supabase.com:6543/postgres"
PORTFOLIOS = "IBKR,CPF,Robinhood"
CURRENCIES = "SGD,USD,EUR"
BASE_CURRENCY = "SGD"
```

4. Reboot the app after saving secrets.

The app is password-protected so your portfolio data is not public. This is a single shared password, not multi-user auth.

## Using the dashboard

### Add a transaction

Open **Raw Transactions → Add Transaction**.

- **BUY / SELL / DEPOSIT / FEE**: enter ticker manually (`NA` for cash deposits/withdrawals).
- **DIVIDEND**: ticker is chosen from holdings with quantity &gt; 0 in the selected portfolio; quantity is prefilled from your open position.

Fields are saved to Supabase. FX conversion and asset metadata (name, asset class) are filled automatically when possible via Yahoo Finance.

### Edit or delete transactions

In **Raw Transactions**, edit cells in the table or remove a row, then click **Apply changes**.

### How holdings are calculated

- **Weighted average cost (WAC)** per portfolio, ticker, and currency.
- **Live prices** from Yahoo Finance for mark-to-market values.
- Amounts are converted to your **base currency** (`BASE_CURRENCY`, default `SGD`) for summary views.

## Customise for your portfolios

Set **`PORTFOLIOS`**, **`CURRENCIES`**, and **`BASE_CURRENCY`** in `.streamlit/secrets.toml`, Streamlit Cloud secrets, or environment variables. No code changes required.

```toml
PORTFOLIOS = "Tiger,MooMoo,Vickers"
CURRENCIES = "SGD,USD,HKD,EUR,JPY"
BASE_CURRENCY = "SGD"
```

If these are not set, the defaults above are used. Portfolio and currency values are validated when you add a transaction.

## Configuration reference

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | Supabase Postgres pooler URL (transaction mode, port 6543) |
| `APP_PASSWORD` | Yes | Login password for the Streamlit app |
| `PORTFOLIOS` | No | Comma-separated portfolio names (default: `Tiger,MooMoo,Vickers`) |
| `CURRENCIES` | No | Comma-separated currency codes (default: `SGD,USD,HKD,EUR,JPY`) |
| `BASE_CURRENCY` | No | Reporting currency (default: `SGD`) |

Secrets can live in `.streamlit/secrets.toml` locally or in Streamlit Cloud secrets. All settings can also be set as environment variables.

## Troubleshooting

**`could not translate host name "db.*.supabase.co"`**  
Use the **pooler** connection string (port `6543`), not the direct database host.

**`DATABASE_URL is not set`**  
Add it to `.streamlit/secrets.toml` or Streamlit Cloud secrets.

**Empty dashboard after setup**  
Add transactions via **Add Transaction**, or load historical data into the `transactions` table (column names are snake_case in Postgres; the app maps them automatically).

**Ticker price or metadata missing**  
Yahoo Finance may not have data for every symbol. Check the ticker format (e.g. `D05.SI` for Singapore listings).
